nav Element

The nav element was first introduced in HTML5. It is a block level element that is used to define a block of navigation links, either within the current document or to other documents. Examples of navigation blocks are menus, tables of contents, and indexes (nav: The Navigation Section element, 2018).

Any links can be contained with a nav element, but it is really only intended for major block of navigation links (HTML nav tag, 2018). Typically, you will find nav elements in navigation bars at the top of a web page. However, the nav element is not placed in the footer tag defining links in the footer of the website (nav: The Navigation Section element, 2018).

Though the implementation of the nav element may seem simple, there is no bonafide consensus of its use. More than that, use of the nav element in your markup has huge implications for users who rely on accessibility technologies like screen readers. User agents, such as screen readers targeting disabled users, can use this element to determine whether to omit the initial rendering of navigation-only content (nav: The Navigation Section element, 2018).

Screen readers use content in nav elements to help users navigate though the website. If the navigation elements are not marked, the screen reader or accessibiliy tool will not be able to locate the navigation elements. So It is critical that you consider what information is contained within your nav elements.
