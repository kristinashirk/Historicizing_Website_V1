Website Navigation

A central theme in web design is the development of a web navigation interface that maximizes usability. A website's overall navigational scheme includes several navigational pieces such as global, local, supplemental, and contextual navigation; all of these are vital aspects of the broad topic of web navigation (Web navigation, n.d.).

Web navigation is difficult because it is so versatile. Every website as a different structure, purpose, and every user has different navigational needs. Designers can bridge the gap between users needs and the design of the website by working to create rich navigational elements, beginning with the nav element. The nav element is helpful to mark the navigational links differently from other links in a website, no matter what navigation structure you use.

Elements of Web Navigation

Information Architecture: Website navigation reflects the underlying structure of the website. If the website is constructed properly, the navigation elements will be effective and easy to use. The complexity of the site’s navigation is ultimately determined by the site hierarchy. Ideally, you want to create an information structure with a flat hierarchy containing relatively few layers, to make navigation quicker and provide a clear, simple path to all the web pages from anywhere on the website.

Global Website Navigation: Global website navigation shows the top level sections/pages of the website. It is available on each page and lists the main content sections/pages of the website. The global navigation elements of your site should reflect the underlying information hierarchy of the site. Global navigation comes in several different styles:

  • Navigation Tabs
  • Navigation Bar
  • Plain List
  • Pull-Down Menu
  • Navigation Buttons
  • Pop-up (cascading) menus (Adkisson, 2005)

Secondary Navigation

Site Map: A site map is a list of pages of a website accessible to crawlers or users. Site maps are typically separate pages within a website that lists all of the pages in a website. They can also be found in the footer of websites.

Site maps provide a visual representation of the information space in order to help users understand where they can go (Nielsen, 2008). Site maps reflect the underlying information architecture of the site. If a site is designed well, the site map can provide the user a useful overview of the website at a glance.

Breadcrumbs: A breadcrumb is a type of secondary navigation scheme that reveals the user’s location in a website or application. Breadcrumbs are usually present in websites that have a large amount of content organized in a hierarchical manner (Gube, 2009). They can also be used as a sort of progress bar for web applications that have more than one step (Gube, 2009).

Typically, breadcrumbs are listed horizontally and separated by the “greater than” symbol (>); the symbol indicates the level of that page relative to the page links beside it (Gube, 2009).

There are many benefits to including breadcrumbs in your sites navigational structure:

  • Breadcrumbs show people their current location relative to higher-level concepts, helping them understand where they are in relation to the rest of the site.
  • Breadcrumbs afford one-click access to higher site levels and thus rescue users who parachute into very specific but inappropriate destinations through search or deep links.
  • Breadcrumbs never cause problems in user testing: people might overlook this small design element, but they never misinterpret breadcrumb trails or have trouble operating them.
  • Breadcrumbs take up very little space on the page. (Nielson, 2007)

Benefits of the nav tag

It is essential that your navigational elements are separated from your content elements. Using the nav tag to sequester navigational elements ultimately helps both the developer and user.

The developer benefits from highly structured content that is easy to navigate visually. If navigational elements are contained in a nav tag, the developer knows right away the purpose and function of the content within the element.

Users benefit from the use of the nav tag with a predictable website structure and accessibility features of the element. Nav elements encourage are a major player in navigation schema standards, and enforce consistent website formatting that makes website navigation intuitive and user friendly.
